# GitHubへのプッシュ手順（初心者向け）

コードを変更した後、GitHubにアップロード（プッシュ）する手順です。

---

## 📋 前提条件

- 以前にGitHubにプッシュしたことがある（リポジトリが接続されている）
- ターミナルが開ける

---

## 🚀 プッシュ手順

### ステップ1: ターミナルを開く

**Macの場合:**
1. 画面の右上の虫眼鏡アイコン（🔍）をクリック
2. 「ターミナル」と入力
3. 「ターミナル」アプリをクリックして開く

**Windowsの場合:**
1. スタートメニューを開く
2. 「cmd」と入力
3. 「コマンドプロンプト」をクリックして開く

---

### ステップ2: プロジェクトのフォルダに移動

ターミナルで以下のコマンドを入力してEnterキーを押します：

```bash
cd /Users/higashiyukako/Documents/cursor/オリジナルシステム開発/tuutihyou.system
```

**確認:**
- 何も表示されなければ成功です
- エラーが出た場合は、フォルダの場所が違う可能性があります

---

### ステップ3: 変更したファイルを確認（オプション）

どのファイルが変更されたか確認したい場合：

```bash
git status
```

**表示例:**
```
Changes not staged for commit:
  modified:   app.py
  modified:   database.py
```

---

### ステップ4: 変更したファイルを追加

以下のコマンドを入力してEnterキーを押します：

```bash
git add .
```

**説明:**
- `.` は「このフォルダの中のすべてのファイル」という意味です
- 変更したファイルをすべて追加します
- 何も表示されなければ成功です

---

### ステップ5: 変更を保存（コミット）

以下のコマンドを入力してEnterキーを押します：

```bash
git commit -m "クラス管理機能を追加"
```

**説明:**
- `commit` は「保存する」という意味です
- `-m` の後は「保存するときのメモ」です
- メッセージは変更内容を簡潔に書きます
  - 例: `"クラス管理機能を追加"`
  - 例: `"バグ修正"`
  - 例: `"UI改善"`

**確認:**
- 「X files changed, Y insertions(+)」のように表示されれば成功です
- 例: `2 files changed, 50 insertions(+)`

---

### ステップ6: GitHubにアップロード（プッシュ）

以下のコマンドを入力してEnterキーを押します：

```bash
git push
```

**説明:**
- `push` は「アップロードする」という意味です
- ファイルをGitHubにアップロードします

**確認:**
- 「Enumerating objects...」のように表示され、最後に「To https://github.com/...」と表示されれば成功です
- 例: `To https://github.com/あなたのユーザー名/tuutihyou-system.git`

---

## ✅ 完了

プッシュが成功すると、Streamlit Cloudが自動的に再デプロイを開始します。

1. **Streamlit Cloudのダッシュボードを確認**
   - https://share.streamlit.io/ にアクセス
   - デプロイしたアプリを選択
   - 「Deployments」タブで再デプロイの進行状況を確認

2. **再デプロイの完了を待つ**
   - 数分かかります
   - 「Running」と表示されれば完了です

3. **アプリを確認**
   - アプリのURLにアクセス
   - 新しい機能が反映されているか確認

## ❓ よくある質問

### Q: 「nothing to commit」と表示される

**A:** 変更したファイルがないか、既にコミット済みです。`git status` で確認してください。

### Q: 「Your branch is ahead of 'origin/main'」と表示される

**A:** コミットは完了していますが、まだプッシュしていない状態です。`git push` を実行してください。

### Q: 「Permission denied」と表示される

**A:** GitHubの認証が必要です。ユーザー名とパスワード（またはトークン）を求められるので、入力してください。

### Q: プッシュしたが、Streamlit Cloudに反映されない

**A:** 
1. Streamlit Cloudのダッシュボードで再デプロイが進行中か確認
2. 数分待ってから再度確認
3. エラーログがないか確認（「Logs」タブ）

### Q: コミットメッセージを間違えた

**A:** 以下のコマンドで修正できます：

```bash
git commit --amend -m "正しいメッセージ"
git push --force
```

## 💡 ヒント

- **定期的にプッシュする**: 変更を加えたら、こまめにプッシュすることをおすすめします
- **コミットメッセージは分かりやすく**: 後で見返したときに、何を変更したか分かるように書きます
- **エラーが出たら**: エラーメッセージをコピーして、検索すると解決方法が見つかることがあります
